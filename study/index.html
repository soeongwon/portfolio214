오답정리

이미지 대체텍스트 제공 방법

  • 의미가 있는 이미지의 경우 이미지에 포함된 컨텐츠를 그대로 기술한다
  • 블릿등의 의미 없는 이미지는 alt속성 자체를 제공하여 사용자로 하여금 의미 없는 이미지임을 인식할 수 있도록 alt=""를 제공한다
  • 이미지맵을 사용하여 이미지로 제공하는 경우 개별 area 요소에 제공하고 각 항목 간의 관계를 숨김 대체 콘텐츠로 제공하거나 longdesc로 대체 콘텐츠를 제공해야 함.
  • 대체텍스트는 간단 명료하게 제공한다

IR효과

  • IR 기법은 이미지 대체텍스트를 제공을 위한 CSS기법으로 다양한 CSS기법을 사용하여 이미지 대체텍스트를 제공할 수 있다
  • Phark Method 기법은 이미지로 대체할 엘리먼트에 배경이미지를 설정하고 글자는 text-indent를 이용하여 화면 바깥으로(-9999px만큼 내어 쓰기) 빼내어 보이지 않게하는 방법이며, 가장 많이 사용하는 기법이다
  • 의미 있는 이미지의 대체 텍스트로 이미지를 off시에도 대체 텍스트를 보여주고자 할 때에는 WA IR 기법을 사용한다
  • 이미지로 대체활 엘리먼트에 배경이미지를 설정하고 글자는 span태그로 감싼 후 z-index:-1을 이용하여 화면에 안보이게 처리하는 기법이 PWA IR기법이다

CSS 속성

  • text-align: justify; 텍스트의 정렬을 양쪽으로 설정한다
  • text-decoration: underlline; 글자 아래에 선을 만든다
  • text-transform: capitalize; 글자의 첫번째 문자만 대문자로 바꿉니다
  • position: absolute; 절대 좌표와 함께 위치를 지정해 줄 수 있습니다.

padding

  • padding-top: 10px; 요소의 위 안쪽 여백을 설정한다
  • padding: 10px 11px 12px 13px; 위쪽/오른쪽/아래/왼쪽으로 여백을 설정한다
  • padding: 10px 11px 12px; 위쪽(10px)/왼쪽,오른쪽(11px)/아래(12px)으로 설정한다

태그

  • 태그는 열린 태그와 닫는 태그가 있다
  • 닫는 태그에는 "/"가 있어야 한다
  • HTML5에서는 "/"가 생략이 가능하다
  • <br>,<img>,<input>,<meta>태그는 닫는 태그가 필요 없으나, <a>태그에는 닫는 태그가 필요하다

DTD

  • DTD 설정을 하지 않으면 브라우저마다 화면이 다르게 나오고 렌더링이 달라진다
  • meta 태그는 웹 문서에 대한 여러가지 정보를 제공합니다.
  • link는 현재 문서와 외부 리소스와의 관계를 명시합니다
  • lang은 문서의 언어를 설정한다

선택자

  • 선택자는 한 번만 사용할 수 있으며, #으로 표시한다
  • elment > elment 태그는 바로 첫번째 자식만 선택한다
  • elment + elment 태그는 바로 옆 형제태그를 선택한다
  • class 선택자는 하나의 태그에 여러 개를 사용할 수 있다

오답정리 19.02.02

스크린리더

스크린 리더는 제품마다 최신술 지원 여부와 동작방식에 일부 차이가 있으므로 2개 정도의 제품으로 중복 테스트를 하는 것을 권장합니다.

한국형 웹 콘텐츠 지침 2.1

  • 이해의 용이성: 콘텐츠는 이해할 수 있어야 한다
  • 운용의 용이성:사용자 인터페이스 구성 요소는 조작 가능하고 내비게이션 할 수 있어야 한다
  • 견고성: 웹 콘텐츠는 미래의 기술로도 접근할 수 있도록 견고하게 만들어야한다

대체텍스트 제공 방법

  • 의미가 없는 이미지는 대체텍스트를 빈 문자로 제공한다
  • 기술적인 요소로 인해 대체 텍스,트를 제공할 수 없는 경우 음성 정보로 대체 수단을 제공할 수 있다
  • 중요도가 낮은 콘텐츠를 이미미지로 제공한 경우 대체텍스트를 동등하게 무조건 제공한다

18pt 이상 크기의 텍스트 콘텐츠 명도 대비 권고 비율
