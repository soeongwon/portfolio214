while 문으로 hello, while! 100번 출력하기


                        while (조건식)
                            실행문;

while문은 조건식이 참인 동안 반복해서 실행문을 실행한다. 실행문이 여러 개라면 중괄호로 묶어도 된다. if 문과 같은 이유로 실행문이 하나이더라도 중괄호로 묶는 것을 권장한다. 


                        while (조건식) {
                            실행문1;
                            실행문2;
                            실행문3;
                        }
                    

                        while (true) {
                            console.log('hello, while!');
                        }

위의 코드를 실행하면 조건식이 참이므로 console.log가 무한히 실행된다 즉, 웹브라우저 콘솔에 무한히 숫자가 찍힌다. 실제로 실행하면 어느 순간 웹 브라우저가 멈춘다. 무한반복을 ㅊ리하느라 웹 브라우저가 다른 일을 할 수 없기 때문. 이때는 웹 브라우저를 강제 종료해야 한다. 이 같은 문제가 발생할 수 있어서 프로그래밍할 때는 무한 반복되는 상황을 방지해야 한다.

반복문을 멈추려면 어떻게 해야 할까? hello, world!를 100번만 출력하고 싶으면, 엎에서도 말했듯이 조건식을 false로 만들어 주면 된다. 그렇다고 조건식에 false를 넣으면 아예 실행문이 실행되지 않는다. 


                        let i = 1;

                        while (i <= 100) {
                            console.log('hello, while!');
                            i++; 
                        }
결과보기

눈 깜짝 할 사이에 hello, while!가 100번 출력된다. 크롬 브라우저는 콘솔에 연이어 같은 값을 출력할 때 일일이 출력하지 않고 왼쪽에 숫자로 몇 번 출력됐는지를 표시한다. 결과로 반환되는 값 100은 i의 값이다.

변수 i를 선언하고 조건식을 i <= 100으로 준다. 사람이 숫자를 세듯이 console.log 할 때마다 i를 1씩 올린다. i = i + 1을 해도 되지만, 간단하게 i += 1이나 i++로 표현할 수 있다. i가 101이 되면 조건식이 false가 되어 반복문이 멈춘다. i에 숫자를 직접 넣어 가며 생각해 보면 이해하기 쉽다.

실제로 프로그래밍할 때는 변수에 첫 번째 값으로 1보다는 0을 많이 넣는다. 프로그래밍에서는 숫자를 0부터 세기 때문 0부터 시작할 때는 조건식을 i <= 99 또는 i < 100으로 바꿔야 한다. 


                        let i = 0;

                        while (i < 100) {
                            console.log('hello, while!');
                            i++; 
                        }

for문으로 반복해서 출력하기

반복문에는 while 문 외에도 for 문이 있다. 


                       for (시작; 조건식; 종료식)
                        실행문;

for 문의 소괄호에는 세 가지 요소가 들어간다. 시작(식과 변수 선언이 들어간다)과 조건식과 종료식이다. 조건식과 실행문은 while문과 같다. 실행문이 여러 개이면 중괄호로 묶어줄 수 있다. while 문보다 복잡하게 보일 수 있지만, 코드를 넣어 보면 그렇지 않다.


                        for (let i = 0; i < 100; i++) {
                            console.log('hello, for!');
                        }
결과보기

콘솔에 hello, for!을 100번 출력하는 코드이다.

시작 부분이 먼저 실행되고 나서 조건식, 실행문, 종료식 순서대로 시행된다. 그 후 다시 조건식을 검사해 만약 조건식이 참이라면 실행문과 종료식을 반복해서 질행한다. 눈치챘을지 모르지만, while문과 유사한 부분이 보인다. while 문에서 흩어져 있던 반복문 요소들을 for문에서는 모아서 표현하고 있다.


                        let i = 0;
                        while (i < 100) {
                            consloe.log('hello, while!');
                            i++;
                        }
                        
                        for (let i = 0; i < 100; i++) {
                            console.log('hello, for!');
                        }

참고로 for문의 시작, 조건식, 종료식은 생략할 수 있다. 따라서 다음과 같은 코드도 가능하다.


                        for (;;) {
                        }
                        (무한 반복)

단, 실제로 코드를 실행하면 무한 반복되어 프로그램이나 브라우저가 멈춰버린다. 조건식이 없어서 반복문이 종료되지 않기 때문이다.

1부터 100까지 출력하기

이번에는 같은 값을 반복해서 출력하는 대신, 다른 값을 연이어 출력해보자. 


                        let i = 0;
                        while (1 < 100) {
                            console.log(i + 1);
                            i++;
                        }
결과보기

i가 0부터 시작하므로 consloe.log(i + 1)을 해야만 1부터 100까지 출력된다. consloe.log(i)를 한다면 0부터 99까지 출력된다.

break문으로 반복문 멈추기

조금 특수한 상황을 생각해 보자. 반복문을 중간에 멈춰야 하는 경우가. 예를 들어, 반복문을 돌릴 대상이 무한히 많을 때가 있다. 자연수나 실수 전체를 대상으로 반복문을 돌리는 경우다. 반복문으로 값을 하나씩 찾다가 원하는 값을 찾으면 반복문을 멈춰야 한다. 멈추지 않으면 반복문이 실행되느라 다음 코드를 실행할 수 없고 찾은 값을 사용할 수 없다.

이럴 때 break문으로 반복문을 멈춘다.


                        let i = 0;
                        while (true) {
                            if (1 === 5) break;
                            i++;
                        }
                        console.log(i);
결과보기

예제의 반복문은 배운 대로라면 while (true)이기 때문에 무한히 반복된다. 하지만 실행문에 break문이 들어 있다는 점을 주목해야 한다. i가 5가 되면 반복문은 종료된다. 따라서 예제의 반복문은 무한히 실행되지 않는다.

무한 반복일 때 for문과 while문 중 어느 쪽이 더 편할까?

무한 반복문을 표현할 때는 보통 while문을 더 많이 사용한다. for문으로도 가능하지만, while문이 더 간결하기 때문이다. 하지만 두 경우 모두 break문이 없으면 무의미하다.


                            // while 문
                            let i = 0;
                            while (true) {
                                console.log(i);
                                i++;
                            }
                            
                            // for문 
                            for (let i = 0; i++) {
                                console.log(i);
                            }

for문은 조건식이 생략되어 무한 반복된다. 하지만 보통은 조건식을 생략하지 않기 때문에 어색해 보일 수 있다. 그래서 무한 반복을 표현할 때는 while문을 더 많이 쓴다.

continue문으로 코드 실행 건너뛰기

가끔 반복문이 특정 조건에서만 실행되기를 원할 수도 있다. 이럴 때는 continue문을 사용한다. continue문을 넣으면 이후 코드는 건너뛰게 된다.


                        let i => 0;
                        while (i < 10) {
                            i++;
                            if (i % 2 === 0) {
                                continue;
                            }
                            consloe.log(i);
                        }
결과보기

예제는 i가 홀수일때 i를 콘솔에 출력하는 코드이다. continue문이 실행되면 그 뒤의 실행문은 실행되지 않으므로 짝수일 때문 continue문이 실행되도록 작성했다. 보통 때와는 다르게 i++가 위에 있는 이유도 생각해보자.

중첩 반복문 사용하기

프로그래밍할 때 어려운 것 중 하나가 반복문 안에 반복문이 들어 있는 경우인데, 이를 중첩 반복문이라고 한다.


                        for (let i = 0; i < 10; i++) {
                            for (let j = 0; j < 10; j++) {
                                console.log(i, j);
                            }
                        }
결과 보기

변수로 i와 j 두 개를 사용했다. 반복문이 중첩됐을 때는 변수의 값이 어떻게 변하는지 추적하는 것이 중요하다.

먼저 i가 0일 때 두 번째 반복문이 실행된다. 여기서 j가 0에서 9까지 증가한 후에 i가 1이되고, 다시 j가 0에서 9까지 증가한다. 이 과정이 반복되어 i가 9가 되고 j가 9가 됐을 때 반복문이 종료된다.

반복문이 두 번 이상 중첩될 수도 있다. 삼중 반복문을 보자. 예제를 조금 더 복잡하게 만들기 위해 반복문 안에 continue문도 넣는다. 변수 i,j,k가 홀수일 때만 콘솔에 출력하도록 햇다. 변수의 값 변화를 잘 생각하면서 결과를 예측해보자.


                        for (let i = 0; i < 5; i++) {
                            if (i % 2 === 0) continue;
                            for (let j = 0; j < 5; j++) {
                                if (j % 2 === 0) continue;
                                for (let k = 0; k < 5; k++) {
                                    if (k % 2 === 0) continue;
                                    consloe.log(i, j, k);
                                }
                            }
                        }
결과 보기

중첩 횟수가 증가할수록 코드도 점점 어려워진다. 실무에서는 대부분 이중 반복문이나 삼중 반복문 정도만 사용하기 때문에 그보다 많이 중첩된 반복문은 사용할 일이 거의 없다.
