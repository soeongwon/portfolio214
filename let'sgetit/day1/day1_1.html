세미콜론

자바스크립트는 하나의 명령이 끝날 때 세미콜론을 붙여도 되고 붙이지 않아도 된다. 다음과 같이 붙이기를 권장함

console.log('hello, world!');

세미콜론을 붙이지 않으면 대부분의 경우 자바스크립트 엔진이 자동으로 세미콜론을 붙여준다. 하지만 일부 명령에서는 가끔 에러가 발생한다. 그러므로 혹시 모를 에러 발생을 방지하고 통일성을 위해 모든 명령 뒤에 세미콜론을 붙이기를 권장한다. 항상 끝에 세미콜론을 붙이는 것을 에러를 방지하는 좋은 습관이다.

한 줄에 여러 명령을 넣을 때는 반드시 명령 뒤에 세미콜론을 붙여 그다음 명령과 구분해야 한다.

console.log('hello, world!'); console.log('hello, javascript!'); console.log('hello');

주석

다른 사람이 작성한 코드를 볼 때 코드에 관한 별도의 설명이 없으면 코드를 이해하기 어려운 경우가 많다. 이럴때 사람만 알아볼 수 있도록(컴퓨터는 무시하고 넘어감.) 설명을 작성한 부분을 주석(comment)이라고 한다. 주석은 코드에 영향을 미치지 않는다. 보토 주석은 코드에 관한 자세한 설명을 작성하거나 특정 코드를 임시로 사용하지 않게 할 때 사용한다.

주석에는 한 줄 주석과 여러 줄 주석 두가지가 있다.

한 줄 주석

주석이 한 줄일 때는 주석 앞에 //를 넣는다.

// hello, comment!출력
                        console.log('hello, comment!');

코드 앞에 //를 붙이면 // 뒤로는 모두 주석이 된다. // 뒤에 나오는 띄어쓰기는 필수는 아니지만, 주석을 보기 좋게 하려고 씀.

다음과 같이 코드 앞에 // 표시를 붙여 코드가 작동하지 않도록 주석 처리 할 수 있다.


                        //console.log('hello, comment!');

// 뒤로는 모두 주석이 된다는 성질을 이용해 코드 뒤에 주석을 작성할 수도 있다.


                        console.log('hello, comment!'); // hello, comment!출력

여러 줄 주석

한 줄로 작성하기에는 주석이 너ㅜ 길 때 여러 줄 주석을 사용한다. 주석 처리 하고 싶은 부분을 /*와 */로 감싼다. 그 사이의 내용은 몇 줄이든 상관 없이 모두 주석이 된다.


                        /* console.log('hello, world!')l
                        console.log('hello, comment!'); */

코드 중간에 여러 줄 주석을 넣을 수도 있다. 이때는 주석을 제외한 나머지 부분이 실행된다.


                        console.log('hello, world!')l /* 여러 줄
                        주석
                        입니다. */ console.log('hello, comment!');

들여쓰기

자바스크립트는 들여쓰기에 제한이 없다 보통 스페이스로 2칸, 스페이스로 4칸, 탭으로 4칸 공백을 두지만, 1칸이나 3칸으로 들여쓰기를 해도 코드 실행에는 아무런 문제가 없다. 들여쓰기는 통일하는게 좋다.


                        if (condition) {
                            console.log('hello world!');
                        }

                        // 위와 같은 코드를

                        if (condition) {
                        console.log('hello, world!');
                        }
                        
                        // 이와 같이 들여쓰기를 하지 않아도 실행하는 데 아무런 문제가 없다.

하지만 들여쓰기를 하는 것만으로도 ㄷ그 부분이 다른 의미가 있는 부분이라는 것을 다른 사람에게 알려줄 수 있어서 코드의 가독성(코드를 읽고 이해하기 쉬운 정도)이 향상된다. 그러므로 들여쓰기를 규칙적으로 허가하기를 권장한다.
